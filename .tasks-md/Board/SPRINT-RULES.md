# Sprint Rules

## Sprint 0

Sprint 0 is the completed foundation baseline before this Jira-style board cleanup. It contains already-completed backend/event/LLM/DOM/frontend/recording/evidence foundation stories.

Sprint 0 does **not** mean Complete LLM Mode is fully complete.

## Sprint 3.5

Sprint 3 is complete. Sprint 3.5 is the current active sprint.

```text
Sprint 3.5 — Backend Contract Confidence, Integration Testing, and Refactor Readiness
```

Only Sprint 2 selected stories live in `Planning`.

## Active sprint Planning rule

`Planning` must contain only stories selected for the current sprint. Do not use `Planning` as a document archive.

## Bugs rule

Bugs are tracked separately from feature stories:

```text
Bugs/Open
Bugs/Blocked
Bugs/Done
```

Every bug fix must create/update a bug ticket before implementation.
